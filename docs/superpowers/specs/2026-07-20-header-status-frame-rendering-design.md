# Enhanced R7 顶栏状态框像素对齐设计

日期：2026-07-20
状态：设计已确认，尚未实现

## 1. 问题与目标

GUI 顶栏的 Profile 与 DualSense 状态控件当前采用全胶囊造型。截图已确认其左右圆弧存在明显阶梯，底边局部还会出现不连续。文字与顶栏其他直线仍然清晰，因此这是 `src/modules/gui/widgets.py` 中 `Pill` 控件的局部几何与 Canvas 绘制问题，不是整个进程被 Windows 位图缩放。

本次目标：

- 将两个顶栏控件改为稳定的小圆角矩形，不再追求半圆胶囊。
- 在 Windows 100%、125%、150%、175% 和 200% 缩放下避免半像素几何。
- 保留现有 Profile、连接方式、电量、充电状态、颜色和多语言信息层级。
- 状态未变化时不重复要求 CustomTkinter 重绘控件。

## 2. 选定方案

继续保留现有 `W.Pill` 调用边界，只修改其内部视觉和更新策略：

- 逻辑高度固定为 `28`。
- 圆角半径固定为 `8`。
- 内部 `CTkLabel` 显式使用与外框一致的逻辑高度。
- 水平 padding、间距和点位尺寸采用 `4` 的倍数，禁止继续使用会在 125% 下形成 `7.5px`、`17.5px` 的值。
- 宽度仍由本地化文本和电量内容自然决定，不设置固定宽度，不截断状态。
- `set_label()`、`set_detail()` 和 `set_dot_color()` 缓存上次呈现值；文本或颜色没有改变时不调用 `configure()`。

关键缩放结果：

| Windows 缩放 | 高度 | 圆角 |
| --- | ---: | ---: |
| 100% | 28px | 8px |
| 125% | 35px | 10px |
| 150% | 42px | 12px |
| 175% | 49px | 14px |
| 200% | 56px | 16px |

所有目标值均为整数像素。该方案比全胶囊更接近现有界面，同时比使用动态图片背景更简单、可靠。

## 3. 范围边界

本次只调整 GUI 顶栏的 Profile 与 DualSense 状态框：

- 不修改全局 Per-Monitor v2、CustomTkinter scaling 或 Windows manifest。
- 不修改导航按钮、设置卡片、更新提示点或其他圆角控件。
- 不修改手柄连接、电量解析、USB/Bluetooth handover、扳机或握把触觉逻辑。
- 不引入 PNG、PIL 动态背景或新的运行时依赖。
- 不重命名对外的 `W.Pill` 类，避免扩大业务代码改动。

## 4. 回退规则

小圆角矩形是唯一首选实现。若在 125% 或 150% 原始分辨率截图中仍出现与当前截图相当的底边断裂或明显圆角阶梯，则只把圆角从 `8` 降为 `0`，变为方框；不继续尝试更复杂的假胶囊或图片绘制方案。

普通小尺寸控件不可避免的一像素边缘过渡不视为失败；与当前截图相同的连续阶梯、底边凹口或局部断线视为失败。

## 5. 验证标准

自动检查：

- 验证顶栏状态框使用 `28` 高度、`8` 圆角以及 4 的倍数间距。
- 验证重复设置相同 label、detail 或 dot color 时不会再次调用底层 `configure()`。
- 现有 controller pill 状态、低电量颜色和语言测试继续通过。

人工检查：

- 分别在 Windows 100%、125% 和 150% 缩放下截取原始分辨率顶栏图像。
- 检查左右圆角、整条底边、Profile 和 DualSense 两种不同宽度。
- 检查“BT / USB”“充电中 30%”“已充满”和较长语言文本不会裁切。
- 最大化、还原窗口后再次检查边缘，确保没有旧 Canvas 残影。

成功条件：三个主要缩放率下均无当前截图所示的明显圆角阶梯和底边不连续；信息、颜色、布局和交互语义保持不变。
