# R4 前端三方案设计

日期：2026-07-16  
状态：经用户授权由 Codex 自审批通过  
适用版本：FH-DualSense-Enhanced R4

## 目标

在不改变触觉、手柄连接、配置保存和 UDP 后端行为的前提下，重新整理 GUI 信息架构，统一使用青绿色主题，并生成三个可独立运行、功能完全一致的 Windows EXE 供审阅。

## 已确认的现状

- GUI 使用 CustomTkinter，入口是 `src/modules/gui/main.py`。
- 业务页面由 `ControlsTab`、`SettingsTab`、`SystemTab`、`ProfilesTab`、`LangTab` 和 `LogsTab` 组成。
- 设置控件直接绑定同一个 `Settings` 实例，并通过 `preferences.save()` 持久化。
- 后端运行在工作线程，Tk 控件只在主线程更新。
- 当前主题集中在 `src/modules/gui/theme.py`，适合扩展成构建时选择的视觉变量。

## 共同信息架构

三个方案共用相同页面模型和设置绑定，不复制业务实现：

1. `总览`
   - 手柄连接状态、传输方式、遥测状态、当前配置、版本和更新状态。
   - 常用功能总开关和一键进入详细设置。
2. `驾驶反馈`
   - 自适应扳机基础阻力、ABS、轮胎抓地力、红线扳机警告、空闲发动机反馈。
3. `握把触觉`
   - 发动机、路面、碰撞、悬挂、轮胎滑移、红线握把反馈和握把换挡冲击。
4. `外观灯效`
   - 转速灯带、红线闪烁和挡位 Player LEDs。所有新功能都有独立开关。
5. `配置`
   - Profile 管理、恢复默认值和语言。
6. `系统与更新`
   - 手柄选择、后台行为、内置更新、DSX 和网络参数。
7. `日志`
   - 日志级别、暂停、清空和复制。

TUI 保持原有页面结构，但新增设置必须提供同等可访问性和翻译，避免配置只能由 GUI 修改。

## 视觉令牌

- 主强调色：`#39C5BB`。
- 深色背景：`#101A1D`、`#142327`、`#193037`。
- 亮色文字：`#EAFDFC`。
- 次强调色：`#86E8E1`。
- 警告色：`#FFCC66`。
- 错误色：`#FF6685`。
- 所有颜色继续从 `src/modules/gui/theme.py` 获取，不在页面中散落硬编码。

主题只使用青绿色视觉语言，不打包角色图片、字体或第三方商标素材。

## 三个可执行方案

### A：Console

- 保留完整左侧文字导航，顶部显示 Profile、DualSense、UDP 和更新状态。
- `总览` 使用两行状态卡和常用开关。
- 设置页面使用单列卡片，风险最低，适合较窄窗口。
- 目标 EXE：`FH-DualSense-Enhanced-R4-console.exe`。

### B：Stage

- 使用顶部横向导航和窄状态带，内容区获得更多横向空间。
- 页面卡片允许双列排列，突出开关与实时状态。
- 小窗口时顶部导航可横向压缩为较短标签，不隐藏功能。
- 目标 EXE：`FH-DualSense-Enhanced-R4-stage.exe`。

### C：Studio

- 使用窄图标轨，右侧工作区带常驻页面标题和状态摘要。
- 长表单使用分组工作台风格，适合高级调参。
- 图标轨提供 Tooltip 和可辨识文字缩写，不能只依赖 Emoji。
- 目标 EXE：`FH-DualSense-Enhanced-R4-studio.exe`。

## 实现边界

- 通过构建变量 `FHDS_UI_VARIANT=console|stage|studio` 选择方案。
- `src/modules/gui/variants.py` 只描述布局和文案元数据。
- 所有方案实例化同一批 Tab 类、共用同一设置对象、后端线程和刷新回调。
- 构建脚本依次构建三个 EXE，并在文件名中体现方案。
- 不能通过复制 `main.py` 或复制设置页面实现三案。

## 验收

- 三个 EXE 都能启动、切换所有页面、保存设置、启动和停止后端。
- 三个 EXE 的设置字段集合一致。
- 窗口在 100%、125% 和 150% DPI 下不出现关键按钮不可见。
- 运行完整测试、`compileall`，并对三个构建执行启动冒烟测试。
