# R4 触觉与 Bluetooth 改进设计

日期：2026-07-16  
状态：经用户授权由 Codex 自审批通过  
参考版本：HorizonHaptics v1.3.0、vDS 0.3.0-rc7、DS5Dongle 当前本地参考快照

## 目标

修复红线握把调节早早饱和的问题，尽可能补齐 HorizonHaptics 中尚未覆盖且不与现有能力冲突的功能，并继续降低 Bluetooth 0x36 模拟 USB 触觉与 USB 48 kHz 音频触觉的感知差异。

## 红线握把改进

当前 `grip_redline_amp / 255 * grip_redline_gain` 在默认值下已经超过 1.0，最终被 `clamp01` 截断。R4 不再把 `gain` 当作唯一强度控制：

- 峰值强度：0 到 255。
- 脉冲频率：1 到 24 Hz。
- 脉冲占空比：20% 到 85%，决定每次脉冲持续时间。
- 低频占比：0 到 1，改变“重量”而不是只放大高频。
- 起始冲击：0 到 1，在进入红线的前 60 到 160 ms 添加可辨识的低频起始包络。
- 背景压低：保留现有逻辑，让红线从路面底噪中突出。

旧 `grip_redline_gain` 保留迁移读取，但 UI 标记为兼容参数；默认配置使用不会饱和的峰值和更高占空比获得约 50% 的主观增强。

## 可补齐能力

### 自适应扳机

1. `增压附加阻力`
   - 使用已解析的 `boost`。
   - 只在 R2 扳机键基础阻力路径叠加，不覆盖打滑、红线或固件墙。
   - 独立开关、阈值和最大附加强度。
2. `G 力油门阻力`
   - 使用横向和纵向加速度合成值，经 EWMA 平滑。
   - 作为基础 R2 扳机键阻力的可选附加项，不能高于抓地力、红线和固件墙优先级。
   - 独立开关、横向权重、纵向权重、满量程 G 和最大附加强度。
3. `碰撞扳机冲击`
   - 复用握把碰撞检测事件，避免各自用不同阈值误报。
   - 默认关闭，L2 和 R2 可分别启用。
   - 优先级高于换挡、ABS、抓地力和基础阻力，持续时间短。
4. `空闲扳机路面纹理`
   - 仅在对应扳机键未被踩下且没有更高优先级效果时生效。
   - 区分 rumble strip 和普通 surface rumble，L2、R2 独立开关。

动态轮胎打滑和 GT7 ABS 墙已经在 `src/modules/forzahorizon/effects.py` 中独立实现，不重复加入第二套算法。

### 外观灯效

1. `转速灯带`
   - 默认关闭。
   - 从指定 RPM 比例开始由青绿色过渡到黄色、红色。
   - 到红线阈值按可调频率闪烁。
2. `挡位 Player LEDs`
   - 默认关闭。
   - 1 挡到 5 挡映射到五颗 LED，空挡和非比赛状态熄灭。
3. 灯效只写 DualSense 标准状态输出字段，USB 和 Bluetooth 共用语义状态。
4. DSX 模式下不写灯光，保持 DSX 对 RGB 的所有权。

## Bluetooth 模拟 USB 改进

已确认的硬限制：

- USB 音频端点使用 48 kHz PCM。
- Bluetooth 0x36 只承载 3 kHz、每包 32 个双声道 int8 帧。
- 当前 0x36 不包含 controller speaker block，也不构成虚拟 USB 设备。

R4 的安全改进：

- 将 PCM renderer 的固定 65/190 Hz 双振荡器改为可选的运输校准表，保留同一 `HapticFrame` 语义。
- Bluetooth 端使用软限幅而不是硬裁剪，减少多层效果叠加后的方波化“傻震”。
- 对 3 kHz 量化增加一阶误差反馈，降低 int8 低幅度细节丢失。
- 增加队列丢包、连续迟到和渲染峰值诊断；出现持续拥塞时自动降低背景层，不丢碰撞和红线事件。
- 保持 398 字节报文、序号、CRC、3 kHz 和 32 帧约束不变。
- 不引入 vDS 驱动，不宣称 Bluetooth 已成为真实 USB 音频设备。

## 冲突和优先级

扳机键优先级：

`碰撞冲击 > 换挡冲击 > ABS/抓地力 > 红线警告 > 固件墙 > 增压/G 力基础阻力 > 普通基础阻力 > 空闲路面纹理`

握把触觉继续使用混合器分层：碰撞事件压低背景，红线事件压低连续路面，灯效不改变触觉层。

## 验收

- 新功能全部有独立开关和数值范围测试。
- 关闭所有新增开关时，R3 的扳机帧和握把帧保持兼容。
- USB 和 Bluetooth 都能发送灯效状态而不破坏扳机和 0x36 PCM。
- 0x36 报告继续固定为 398 字节并通过 CRC 测试。
- Bluetooth 量化器和软限幅使用确定性单元测试。
- 需要真实手柄最终审阅的项目明确列出，不以单元测试代替手感结论。
