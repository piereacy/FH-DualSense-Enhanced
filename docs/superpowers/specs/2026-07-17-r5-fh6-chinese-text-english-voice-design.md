# Enhanced R5 FH6 中文文字与英文语音按钮设计

## 目标

在 Windows Steam 版 Forza Horizon 6 中提供显式按钮，让用户把游戏配置为中文文字与英文语音，并可通过另一按钮恢复原始语言文件。实现沿用用户已验证的手动方法：在 Steam 游戏语言保持英文时，交换 `media\Stripped\StringTables` 中 `CHS.zip` 与 `EN.zip` 的文件名。

应用只负责发现安装目录、检查前置条件、执行可回滚的三步改名和展示当前状态。路径发现可以自动进行，但任何交换、恢复或残留状态修复都必须由用户点击按钮并确认，普通启动绝不自动修改游戏文件。

## 已验证的本机事实

- Steam App ID 为 `2483190`，manifest 名称为 `appmanifest_2483190.acf`。
- 用户当前安装目录由 Steam 记录为 `C:\Program Files (x86)\Steam\steamapps\common\ForzaHorizon6`，但该路径只用于本机调查，不进入生产默认值。
- `CHS.zip` 与 `EN.zip` 都是可读取 ZIP，各包含 288 个 `.str` 条目；抽样内容分别为简体中文与英文。
- 当前 Steam manifest 的游戏语言为 `english`。
- 当前 `StringTables` 目录 ACL 允许普通 `BUILTIN\Users` 修改，因此本机不需要管理员权限。应用仍必须正确报告其他机器上的权限错误，且不自动提权。

这些事实证明用户描述的方法与当前安装匹配，但不能替代其他用户机器上的动态发现和运行时验证。

## 支持边界

- 首版只支持 Windows Steam 版 FH6。
- 不支持 Microsoft Store、Xbox App、未知重打包版本或 Linux/Proton 路径修改。
- 不修改 Steam manifest、Steam 游戏语言、注册表或游戏可执行文件。
- 不复制、覆盖、删除或分发游戏资产，只在同一目录内改名现有的两个 ZIP。
- 不在 FH6 运行时改名。
- 应用只能确认 Steam 英文配置和字符串文件交换成功，不能自动验证游戏内语音播放结果。

## 安装目录发现

生产代码不得硬编码盘符或 `Program Files`。发现顺序如下：

1. 从用户和系统注册表读取实际 Steam 根目录。
2. 解析该根目录下的 `steamapps\libraryfolders.vdf`，枚举所有 Steam 库。
3. 在声明包含 App `2483190` 的库中读取 `steamapps\appmanifest_2483190.acf`。
4. 使用 manifest 的 `installdir` 生成 `<library>\steamapps\common\<installdir>`。
5. Steam 配置发现失败时，读取 `Steam App 2483190` 卸载项的 `InstallLocation`。
6. 注册表也失效时，在 FH6 运行期间从 `psutil` 捕获 `ForzaHorizon6.exe` 的完整路径。
7. GUI 最后提供游戏根目录选择器，TUI 提供路径输入框。

候选路径必须解析为绝对路径，并同时满足：

- 根目录存在 `ForzaHorizon6.exe`。
- 相对位置 `media\Stripped\StringTables` 存在。
- 该目录中存在两个可读取的常规文件 `CHS.zip` 与 `EN.zip`，或存在可安全修复的专用临时状态。
- 所有待改名路径的解析后父目录都等于已验证的 `StringTables` 目录。

成功发现的 FH6 根目录缓存为 global 本地设置，例如 `fh6_install_path`。该字段加入 `preferences.GLOBAL_FIELDS`，不进入 Profile、命名快照或分享码。每次执行按钮操作前仍重新验证实际路径和文件，不盲信缓存。

## Steam 语言检查

当 appmanifest 可读时，读取 `UserConfig` 或 `MountedConfig` 中的 `language`：

- `english`：允许继续。
- 明确为其他语言：禁用交换按钮，提示用户在 Steam 游戏属性中改成英文。
- manifest 不可读或语言未知：显示未确认警告。用户只有在确认对话框中明确确认 Steam 已设为英文后才能继续。

应用不写 appmanifest，也不要求用户为路径发现而运行一次游戏。运行一次 FH6 只是 manifest、卸载注册表和库配置都无法定位时的进程路径后备方案。

## 文件状态识别

状态判断以磁盘当前内容为准，不依赖内存布尔值。模块从多个稳定的小型 `.str` 条目抽样，分别确认明显的简体中文内容和英文内容。任何分类不明确的结果都返回未知，不猜测改名。

主要状态为：

- 原始状态：`CHS.zip` 是中文内容，`EN.zip` 是英文内容。
- 已交换状态：`CHS.zip` 是英文内容，`EN.zip` 是中文内容。
- 需要修复：专用临时文件存在，且预期两个语言内容仍可唯一识别。
- 未知：文件缺失、ZIP 损坏、存在多余冲突文件、内容无法分类或组合不符合以上状态。

Steam 更新或“验证文件完整性”可能恢复原始命名。页面每次刷新都重新读取实际状态，因此会显示原始状态，不自动重新交换。

## 按钮式交换

原始状态下显示“启用中文文字 + 英文语音”。用户确认后执行：

1. `CHS.zip` 改名为 `CHS.zip.fhds-swap.tmp`。
2. `EN.zip` 改名为 `CHS.zip`。
3. `CHS.zip.fhds-swap.tmp` 改名为 `EN.zip`。

已交换状态下显示“恢复原始语言文件”。恢复使用同一三步交换，因此中文内容重新位于 `CHS.zip`，英文内容重新位于 `EN.zip`。

实现使用不覆盖既有目标的 rename 语义，不使用 `replace`。每一步前确认源存在、目标不存在、FH6 仍未运行，并确认相关路径仍在同一个已验证目录内。操作在后台线程执行，期间禁用全部语言文件按钮；Tk 控件只由主线程更新。

## 失败、回滚与残留修复

模块记录当前函数调用内完成到哪一步。发生权限错误、文件占用、目标冲突或其他异常时，立即按现有文件组合逆向改名，尽力恢复按钮操作前的状态。回滚失败时保留专用临时文件和完整日志，不删除任何 ZIP。

若进程被强制结束并留下临时文件，下次启动只读检测为“需要修复”。应用不得自动修复。用户点击“修复语言文件”并确认后，模块根据 ZIP 内容把中文归位到 `CHS.zip`、英文归位到 `EN.zip`，最终恢复原始状态。只有恰好两个可识别语言归属的文件时才允许修复；其他组合保持未知并提示使用 Steam 验证文件完整性。

所有错误在界面显示简短、可操作的说明，完整路径和异常写入日志。应用不请求管理员权限，也不以强制覆盖绕过访问控制。

## GUI 与 TUI

GUI 的“系统与更新”页新增“FH6 语言组合”卡片，不新增导航页。卡片包含：

- 当前发现状态和已验证的游戏路径。
- 原始、已交换、游戏运行中、Steam 非英文、需要修复或未知状态说明。
- “启用中文文字 + 英文语音”或“恢复原始语言文件”主按钮。
- 需要时显示“修复语言文件”。
- “重新检测”和“选择游戏目录”按钮。

GUI 的改名动作使用确认弹窗。TUI 系统页提供同等状态、按钮和路径输入框，破坏性按钮使用二次点击确认。GUI 与 TUI 都只自动执行发现和刷新，绝不自动调用交换、恢复或修复。

## 与现有进程检测的关系

现有 `ProcessWatcher` 已经读取进程名和 `exe`，但只返回名称。将进程查找拆出为可复用的只读结果，使退出检测继续使用名称，FH6 路径发现可以读取完整 EXE 路径。该重构不得改变现有“见到 Forza 后消失才退出”的行为、轮询频率或异常容错。

路径捕获成功后立即保存经过验证的游戏根目录。由于默认退出规则可能在 FH6 关闭后结束应用，用户可以重新打开应用，再点击语言交换按钮。

## 测试

### 路径发现

- Steam 安装在非 C 盘。
- Steam 根目录和游戏库位于不同盘符。
- 多个 `libraryfolders.vdf` 库，只有其中一个包含 App `2483190`。
- 自定义 `installdir`。
- 卸载注册表、运行进程和手动路径回退。
- 缓存路径失效后重新发现。
- 不存在任何固定 `C:` 或 `Program Files` 生产默认值。

### 文件状态与交换

- 使用小型 ZIP fixture 覆盖原始、已交换、缺失、损坏、未知和残留临时状态。
- 正向交换和恢复后两个语言内容均保持完整。
- 分别在三次改名的每一步注入失败，确认回滚后没有丢失任一文件。
- 回滚失败时保留可诊断临时状态，不覆盖目标。
- FH6 运行中、Steam 明确非英文和路径验证失败时不发生改名。
- 普通启动、自动发现、页面刷新和残留检测均不调用文件改名。
- 只有确认后的 GUI/TUI 按钮处理器可以调用交换、恢复或修复。

### 界面和项目验证

- GUI/TUI 功能和全部非英语语言目录同步。
- global 路径字段不进入 Profile 或分享码。
- 运行相关定向测试、`uv run --project src pytest -q`、`python -m compileall -q src/modules src/lang` 和 `git diff --check`。
- 启动源码 GUI 验证本机路径可被只读发现，但自动测试和开发冒烟不得交换真实 FH6 文件。
- 真实按钮交换与游戏内中文文字、英文语音结果由用户执行后记录；未执行时必须写“未执行”。

## 非目标

- 不自动切换语言文件，不在 FH6 启动或退出时自动应用。
- 不自动修改 Steam 游戏语言。
- 不备份、复制、删除或在线下载游戏 ZIP。
- 不支持任意游戏或任意语言组合。
- 不绕过 Steam 文件校验、反作弊、权限或文件锁。
- 不把“文件交换成功”描述为已经验证真实英文语音。
