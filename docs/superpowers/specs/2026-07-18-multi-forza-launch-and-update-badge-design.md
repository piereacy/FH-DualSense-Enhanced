# FH4/FH5/FH6 Steam 启动与更新提示点设计

日期：2026-07-18

状态：设计已确认，待实现

## 1. 目标

在现有 Windows GUI 的总览快捷入口中，将只支持 FH6 的启动按钮扩展为支持 Steam 版 Forza Horizon 4、Forza Horizon 5 和 Forza Horizon 6 的分体按钮，同时修复“系统与更新”导航项上更新提示点出现黑色方块底色的问题。

本次只扩展启动入口、用户文案和与之直接相关的文档。FH6“中文文字 + 英文语音”仍是独立的 FH6 专属功能，不扩展到 FH4 或 FH5。

## 2. 已确认的产品行为

### 2.1 支持范围

- 仅支持 Windows Steam 版 FH4、FH5 和 FH6。
- Steam App ID 分别为 FH4 `1293830`、FH5 `1551360`、FH6 `2483190`。
- 不支持 Xbox App 或 Microsoft Store 安装，本次也不探测它们。
- 继续通过 `steam://run/<app-id>` 交给 Steam 启动，不直接执行游戏 EXE，不自动提权。
- 启动始终由用户点击触发，程序不得自动启动任何游戏。

### 2.2 分体启动按钮

- 总览快捷入口显示一个分体按钮：左侧为主启动按钮，右侧为窄箭头按钮。
- 左侧按钮启动当前选中的游戏；右侧箭头展开 FH4、FH5、FH6 三个选项。
- 选择菜单项只改变当前选择，不立即启动游戏。
- 第一次启动软件时默认选择 FH6；用户选择其他游戏后持久保存，下次启动恢复最后选择。
- 未安装的游戏仍可被选择，不自动回退到其他已安装游戏。
- 当前游戏未发现时，左侧按钮禁用并明确显示 `FH4 未找到`、`FH5 未找到` 或 `FH6 未找到`。
- 扫描、启动中和运行中的状态文案都必须带当前游戏简称，避免用户不知道按钮正在处理哪个游戏。
- 切换选择会立即开始该游戏的后台安装发现。旧选择尚未返回的异步结果必须被丢弃，不能覆盖新选择的状态。
- 提交 Steam 启动请求期间暂时禁用游戏选择；请求提交完成后恢复。已经交给 Steam 的启动请求不能被软件撤回。

### 2.3 命名边界

- 面向用户的通用标题、说明和故障提示使用 `Forza Horizon`，不再把整个项目或通用启动能力称为 FH6 专用。
- 与当前选择相关的动态状态保留 FH4、FH5、FH6 简称。
- FH6“中文文字 + 英文语音”、FH6 语言包路径和 FH6 专属模组提醒继续明确写 FH6，不做泛化。
- TUI 当前只有 FH6 语言工具，没有游戏启动入口。本次不新增 TUI 启动器；只修正其中确实属于通用产品范围、却误写为 FH6 的文案。

## 3. 代码结构

### 3.1 通用 Steam 游戏注册表

新增 `src/modules/forzahorizon/game_launch.py`，集中保存不可变游戏定义：

- 稳定键名：`fh4`、`fh5`、`fh6`。
- 用户简称和完整名称。
- Steam App ID。
- 精确可执行文件名：`ForzaHorizon4.exe`、`ForzaHorizon5.exe`、`ForzaHorizon6.exe`。

该模块提供以下通用职责：

- 根据显式路径提示和 Steam library/appmanifest 发现安装目录。
- 验证路径中的游戏 EXE 与所选游戏精确匹配。
- 用精确进程名判断所选游戏是否运行。
- 生成并提交对应的 Steam URI。
- 对非法游戏键执行明确的校验失败，不悄悄选择另一款游戏。

安装发现保留现有 FH6 代码已经覆盖的 Steam 注册表、`libraryfolders.vdf` 和 `appmanifest_<app-id>.acf` 路径。通用逻辑从 FH6 语言模块中提取，避免复制三份实现。

### 3.2 FH6 语言功能隔离

`src/modules/forzahorizon/fh6_language.py` 改为复用通用模块的 FH6 安装发现和运行检测，然后继续执行自身的专属校验：

- `media/Stripped/StringTables` 存在。
- 语言 ZIP 内容有效。
- Steam 当前为 English。
- 游戏未运行。
- 交换、恢复和崩溃恢复仍使用原有事务边界。

通用启动模块不得导入语言包交换逻辑。点击启动游戏不得隐式修改、修复、恢复或删除任何语言文件。

### 3.3 配置与持久化

在 `src/modules/config/settings.py` 增加：

- `preferred_forza_game: str = "fh6"`。
- `fh4_install_path: str = ""`。
- `fh5_install_path: str = ""`。

继续保留现有 `fh6_install_path`，供 FH6 启动发现与 FH6 语言功能共同复用。上述字段加入 `preferences.GLOBAL_FIELDS`，属于全局软件偏好，不进入车辆 Profile 或分享码。

恢复出厂设置时，选择恢复为 FH6，三个安装路径提示恢复为空，再由发现逻辑重新扫描。旧配置没有新字段时，由 dataclass 默认值无损迁移。

### 3.4 GUI 状态与线程边界

`src/modules/gui/overview_tab.py` 将 FH6 单例状态改为按游戏键缓存的安装状态，并保存当前选择。安装扫描和进程检查仍在后台短任务中执行；Tk 控件只由主线程更新。

每个异步扫描结果携带游戏键和请求序号。主线程只接受与当前选择和最新序号都一致的结果。这样用户快速切换 FH4、FH5、FH6 时，较慢的旧扫描不会把按钮状态改回旧游戏。

启动状态机包含：

1. 正在查找所选游戏。
2. 已安装，可以启动。
3. 未找到，按钮禁用。
4. 正在向 Steam 提交启动请求。
5. 等待游戏进程出现。
6. 游戏正在运行，按钮禁用。
7. 启动超时或 URI 提交失败，显示具体游戏名并写日志，然后回到可重试状态。

通用状态纯函数放在 `src/modules/gui/overview_status.py`，便于不启动 GUI 就覆盖文案与启用状态测试。

## 4. 更新提示点修复

当前更新提示点是导航按钮外层容器的子控件，透明背景解析到了深色侧边栏，因此在选中态的青绿色按钮上显示为黑色方块。

修复方式：

- 将提示点作为“系统与更新”导航按钮本身的子控件。
- 提示点控件显式使用透明背景，仅绘制白色圆点。
- 保持现有点击行为，点击提示点仍进入“系统与更新”页面。
- 验证导航项选中、未选中和 hover 三种状态，不能出现黑色或其他矩形底色。

不引入图片资源或新的主题色，避免 DPI 缩放产生模糊位图。

## 5. 用户文档与长期文档

- `README.md`、`docs/ReadmeZH.md`、`docs/ReadmeJA.md` 在支持范围中明确写 FH4、FH5、FH6。
- README 保持精简，只写用户可感知的三游戏支持、Windows Steam 限制、Steam Input 开启和游戏内振动关闭等必需信息，不展开内部扫描算法。
- 通用项目描述中的 FH6 改为 Forza Horizon；FH6 语言功能和 FH6 模组提醒保持原名。
- “老三样”即 `AGENTS.md`、`docs/ARCHITECTURE.md`、`docs/DECISIONS.md`，全部记录：今后修改游戏启动、进程检测、安装发现、更新说明或相关用户文案时，必须同时考虑 FH4、FH5、FH6，除非功能被明确标记为某一代专属。
- 因代码状态与验证结果将变化，完成实现后按既有独立规则同步 `docs/PROJECT_STATE.md`，但不把它称为“老三样”。
- 若 Release 工作流或当前版本说明仍把通用启动能力写成只支持 FH6，同步修正为三游戏支持；Release 正文继续保持完整中英双语。

## 6. 错误处理与日志

- Steam 不可用、URI 提交失败、安装清单损坏和路径校验失败分别记录可诊断日志。
- GUI 主文案保持简短，详细路径和异常只进入日志。
- 某一游戏发现失败不影响另外两款游戏的缓存和可用性。
- 进程检测使用精确 EXE 名，不能只用 `forza` 子串决定启动按钮状态。
- 现有“游戏关闭后同时关闭软件”可继续使用通用 Forza 进程观察规则；本次只保证 FH4、FH5、FH6 都在其考虑范围内，不改变用户已有后台行为选项。

## 7. 测试与验收

### 7.1 自动测试

- 游戏注册表包含三个正确 App ID 和精确 EXE 名。
- 临时 Steam library 与 appmanifest 样本可以分别发现 FH4、FH5、FH6。
- 显式路径提示必须与所选游戏 EXE 匹配。
- 精确进程检测不会把另一代游戏或无关 Forza 进程判为当前游戏。
- 每个选择生成正确的 `steam://run/<app-id>`。
- 首次配置默认 FH6，改变后跨重启保留，恢复出厂设置回到 FH6。
- 全局选择和路径不会进入命名车辆 Profile 或分享码。
- 分体按钮菜单只选择不启动，左侧按钮只启动当前选择。
- 未安装状态禁用主按钮且不自动回退。
- 快速切换时丢弃旧扫描结果。
- 所有语言模块具备新增键或明确回退到英文。
- 更新提示点挂在导航按钮上且背景透明。
- 三语 README 与老三样的三游戏契约通过。
- 现有 FH6 语言交换、恢复和路径发现测试继续通过。

### 7.2 手工 GUI 验收

- 在 Windows 默认缩放和用户当前缩放下检查分体按钮宽度、文字截断、箭头菜单位置和鼠标滚轮行为。
- 依次选择 FH4、FH5、FH6，确认选择动作不启动游戏，重新打开软件后恢复最后选择。
- 对未安装游戏确认按钮显示“未找到”且禁用，不自动切换。
- 不实际启动游戏的常规冒烟测试之外，若执行真实启动验收，只启动用户明确选择的一款 Steam 游戏。
- 检查更新提示点在导航项选中、未选中和 hover 时均为透明背景白点。

### 7.3 完成前检查

执行与改动相关的定向测试、完整 `pytest`、`compileall`、`git diff --check` 和完整工作区差异复核。任何未执行的真实 Steam 启动或 GUI 视觉验收必须明确写为“未执行”，不能推断为已通过。

## 8. 不在本次范围

- Xbox App、Microsoft Store 或 Game Pass 安装发现和启动。
- FH4/FH5 的语言包替换。
- 自动选择已安装游戏或自动回退。
- 自动启动游戏。
- 新增 TUI 游戏启动器。
- 改变 DualSense 遥测、扳机、握把触觉或灯效算法。
- 为本次改动提前确定新的 Enhanced 版本号或发布标签。
